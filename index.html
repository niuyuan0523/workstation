部署架构说明

如何访问子项目？

每个子项目都部署在仓库的子目录中。例如：/project-a/。这是因为 GitHub Pages 对于非 user.github.io 仓库，默认将仓库名作为根路径。

关键技术点

  • Vite/Webpack 配置了 base: '/repo-name/project-name/'
  • React Router/Vue Router 使用了 History 模式并设置了 basename
  • 通过 GitHub Actions 自动构建并合并到 gh-pages 分支
